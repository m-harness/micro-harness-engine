# Admin Guide

管理画面 (`/admin`) の全機能解説です。

---

## アクセス

```
http://localhost:4310/admin
```

### ログイン

- **Login Name**: `root`（またはadminロールのユーザー）
- **Password**: `ADMIN_RUNTIME_PASSWORD` 環境変数の値

### 認証の特徴

| 項目 | 詳細 |
|---|---|
| セッション保存 | メモリ（DBに保存しない） |
| 有効期限 | デフォルト12時間（延長なし） |
| サーバー再起動 | セッション消失（再ログイン必要） |
| CSRF | 常に必須（全mutating操作） |

---

## Overview

ダッシュボードでシステムの概要を確認します。

| 項目 | 説明 |
|---|---|
| ユーザー数 | 登録ユーザーの総数 |
| 承認待ち | pending状態の承認リクエスト数 |
| Automation数 | 登録された定期実行タスク数 |
| Tool Policy数 | 作成済みTool Policy数 |
| File Policy数 | 作成済みFile Policy数 |
| ツールカタログ数 | 利用可能なツールの総数 |
| 保護ルール数 | 有効な保護ルールの数 |

## Users

### ユーザー作成

| フィールド | 制約 |
|---|---|
| Login Name | `[a-z0-9][a-z0-9-_]{1,31}`, ユニーク |
| Display Name | 任意の文字列 |
| Password | 12文字以上 |
| Role | `user` / `admin` |

### ユーザー管理

| 操作 | 説明 |
|---|---|
| 編集 | loginName, displayName, role, status を変更 |
| パスワード変更 | 新しいパスワードを設定 |
| 無効化 | status を `inactive` に変更（ログイン不可） |
| 有効化 | status を `active` に戻す |
| 削除 | ユーザーを完全削除（関連データもカスケード削除） |
| ポリシー割り当て | Tool Policy と File Policy を選択 |

### rootユーザー

`root` はシステムが自動作成する特殊ユーザーです:

- 削除不可
- 無効化不可
- パスワード変更不可（環境変数で管理）
- `authSource: system`
- `System All Tools` ポリシーが割り当て済み

## Tool Policies

### 一覧表示

各ポリシーの名前、説明、許可ツール数、ツール詳細（名前・リスクレベル）を表示します。

### 作成

1. ポリシー名と説明を入力
2. 登録済みツール一覧から許可するツールをチェック
3. MCP ツールも同じ画面で選択可能

### 編集

- 名前・説明の変更
- 許可ツールの追加・削除
- **システムポリシーは編集不可**

### 削除

- 割り当て中のユーザーがいる場合、代替ポリシーを指定
- 該当ユーザーは自動的に代替ポリシーに移行
- **システムポリシーは削除不可**

---

## File Policies

### 一覧表示

各ポリシーの名前、説明、ルート一覧（scope, rootPath, pathType）を表示します。

### 作成

1. ポリシー名と説明を入力
2. ルートを追加

### ルートの追加

| フィールド | 選択肢 | 説明 |
|---|---|---|
| Scope | `workspace` / `absolute` | パスの基準 |
| Root Path | 文字列 | アクセスを許可するパス |
| Path Type | `dir` / `file` | ディレクトリ全体 or 単一ファイル |

`absolute` スコープのルートは、パスが実在する必要があります。追加前に **Probe** 機能で確認できます。

### Probe

パスの状態を事前確認する機能です:

- 入力したパスの絶対パスへの解決結果
- 存在するか
- ワークスペース内か
- ディレクトリかファイルか

### ルートの削除

個別のルートを削除できます。システムポリシーのルートは削除不可。

## Approvals

### 一覧

pending 状態の全承認リクエストを表示します。

| 表示項目 | 説明 |
|---|---|
| ツール名 | 承認が要求されたツール |
| 理由 | 承認が必要な理由 |
| リクエスト者 | 操作を要求したユーザー |
| リクエスト日時 | いつ要求されたか |

### 操作

- **Approve**: 操作を承認し、ツール実行を再開
- **Deny**: 操作を拒否し、LLMに拒否結果を返す

管理者による承認は `decided_by_user_id: null`、`decision_note: 'admin-console'` として記録されます。

## Automations

### 一覧

全ユーザーのAutomationを表示します。

| 表示項目 | 説明 |
|---|---|
| 名前 | Automationの名前 |
| オーナー | 作成したユーザー |
| 命令 | 実行する内容 |
| 間隔 | 実行間隔（分） |
| ステータス | active / paused / deleted |
| 次回実行 | 次の実行予定日時 |
| 前回実行 | 最後の実行日時 |

### 操作

| 操作 | 説明 |
|---|---|
| 一時停止 | Automationを一時停止（オーナーの確認なし） |
| 削除 | Automationを削除（オーナーの確認なし） |

管理者の操作はオーナーシップチェックをバイパスします（緊急時の操作を想定）。

## Tools

登録されたツールのカタログ一覧です。

| 表示項目 | 説明 |
|---|---|
| 名前 | ツール名 |
| 説明 | ツールの説明 |
| リスクレベル | `safe` / `dangerous` |
| プラグイン名 | 所属プラグイン or `mcp` |

MCPツールは `servername__toolname` 形式で表示されます。

## Skills

カスタムスキルの管理です。スキルはMarkdownファイルとして `skills/` ディレクトリに保存されます。

### 作成

| フィールド | 制約 | 説明 |
|---|---|---|
| Name | `[a-z0-9_]`, 最大64文字 | スキル識別名 |
| Description | 必須 | LLMに表示される説明 |
| Prompt | 必須 | スキル実行時にLLMに渡されるプロンプト |

### 編集

説明とプロンプトを更新できます。

### 削除

スキルファイルを削除します。

### 動作の仕組み

1. LLMは `use_skill` ツールを通じてスキルを呼び出す
2. スキルのプロンプトがLLMに返される
3. LLMはそのプロンプトに従って行動する

---

## MCP Servers

外部MCPサーバーの接続管理です。

### 一覧

| 表示項目 | 説明 |
|---|---|
| サーバー名 | 設定した名前 |
| 設定 | command/url（envとheadersはマスク） |
| 状態 | `ready` / `connecting` / `disconnected` / `failed` |
| ツール数 | 提供されるツールの数 |
| 最終エラー | 最後のエラーメッセージ |

### 追加

stdio の場合:
```json
{
  "command": "npx",
  "args": ["-y", "server-package"],
  "env": { "API_KEY": "xxx" }
}
```

HTTP の場合:
```json
{
  "url": "https://mcp.example.com/sse",
  "headers": { "Authorization": "Bearer xxx" }
}
```

### 操作

| 操作 | 説明 |
|---|---|
| 編集 | 設定を変更（再接続が発生） |
| 削除 | サーバーを切断・設定を削除 |
| 再接続 | 切断されたサーバーに再接続 |

## Protection Rules

パス保護ルールの一覧表示です。

### 表示内容

| 項目 | 説明 |
|---|---|
| パターン | マッチングパターン（`.env`, `security`, `**/*.key` 等） |
| 種別 | `exact` / `dirname` / `glob` |
| スコープ | `system` / `workspace` |
| 効果 | `deny` |
| 優先度 | 数値（小さいほど優先） |
| 有効/無効 | トグル可能 |

### 操作

| 操作 | 説明 |
|---|---|
| 追加 | 新しい保護ルールを作成 |
| 有効/無効切り替え | ルールの一時的な無効化 |
| 削除 | ルールを削除（`system` スコープは管理画面からは操作制限あり） |
| 検査 | 特定のパスがどのルールにマッチするかテスト |

### 検査（Inspect）

パスを入力すると、どの保護ルールにマッチするかを確認できます。ルール設定のデバッグに使用します。
